Description

The learning outcomes of this assignment are:

  1. Ability to design and implement classes in C++, using the g++ compiler. Here is Juan Soulie's tutorial.jar which covers the basics of C++.
  2. Knowledge working with the Ant build tool and the command-line and a text editor for developing C++ applications.
  3. Experience working with C++ Full Light ToolKit (FLTK) user-interfaces and their components; especially simple controls that do not require an underlying data model or data source. Handling events generated by FLTK components.
  4. Experience creating C++ programs having multiple classes
  5. Experience using pre-compiled auxilary C++ classes contained in library archive.
  6. Experience working with Serialization using Json.

The pre-requisites for this assignment are:

  1. Units 1-4 of the Ser321 course content. This includes Linux, C++, Ant, UI FLTK, and Json Serialization
  2. The Development Environment for this assignment includes VirtualBox Linux with the packages ant, g++, and libjsoncpp package. Note that Ant must be augmented with the cpptasks.jar as explained in the readme.txt file of the C++ example: cppFraction.jar.

In this assignment, create a non-distributed Music Browser application in C++ that utilizes the instructor provided FLTK GUI. The Music Browser is a collection of Album objects, similar to that created in the two prior Java assignments. You are to create Album, and Track classes, that serializes/deserializes to/from the same text file you produced in the preceding assignment, which is in Json format. Note that you should initiate your application with the same json file as used for the Java version of this assignment. The user interface provides the ability for browsing albums that are in the collection.

The browser's constructor should read in the file initializing the (Music Library) to include a collection of albums. Your Album and Track classes should include a constructors to initialize an object from a Json::Value, and another method to export an object to a Json::Value. Be sure to include an appropriate initialization file with the solution you submit.

The user interface for your client is provided in the example assign4WebDist.jar. It includes a readme.txt file explaining how to run the sample. The included Ant build file will compile and execute a sample client. It depends on three libraries that you should install on your development environment (see the class resources pages and the readme.txt file in the sample). The sample includes src/sample/MediaClientGui.cpp provides the instructor provided MediaClientGui class, which is the View for your solution. There is also a sample main program demonstrating the use of the GUI with C++ in the program src/sample/samplemain.cpp. This is an example Controller for the View.

Another example project that demonstrates serialziation to/from Json for both C++ and Java is provided in the example groupJson.jar.

Below is a view of the MediaClientGui. You should use this FLTK GUI for solving this assignment.


In summary, create a stand-alone Music Browser Client application, using C++/FLTK. Your application should initialize a music library from a Json file, and have the ability to select, remove, and add albums and tracks within albums. You should also support exporting the library of Albums using Json serialization to a file. Future assignments will enhance this solution by distributing it using JsonRPC.

Functionality. Your solution should provide the same functionality as the stand-alone Java Music Browser application you completed in prior assignments

Your C++ files should contain class comments similar in structure to that of the instructor's example code. This constraint applies to all source code you submit in this course. The comments should include a right-to-use license that provides the course the right to compile, run, and evaluate the software for purposes of course grading and program (BSSE) assessment. The comments should also include your copyright, your name, purpose, and your contact information (email), as well as software version and modification/creation date.

The Music Library Interface

To make future assignments easier, your solution should include three classes for the underlying application Model: Album, Track, and MusicLibrary. These classes should have constructors for Json::Value, and a method to export and object's state to Json::Value. Your MusicLibrary class may also support methods similar to those defined in the Java version of this assignment. The next assignment will create a JsonRpc server for your MusicLibrary.

The following examples should provide the basis for constructing your solution.

  • guiDemo.jar      This project demonstrates use of Model, View, and Control (MVC) separation of concerns for Windowed applicaitons in Java with Swing and C++ with FLTK

Structure of your Solution Directory

Submit your solution on Canvas as a Java archive file (jar) of the project directory: Assign4. Grading your project will be done by extracting the jar and executing the project using Ant from the command-line on Virtual Box Debian as defined as the Course Development Environment. The project directory should include only the following files, subdirectories and classes:

  • The Ant directive file build.xml. The build should contain targets for targets, clean, prepare, build, and execute. The execute target compiles and executes your C++ Music Browser with appropriate arguments. Other examples are covered in class to provide Json serialization.
  • Subdirectory: src/client for your solution's source files.
  • Subdirectory bin used to store the executable program as generated by your build target. The bin folder may be generated by Ant.
  • A file albums.json which contains a Json initialization file for your music library.
